# コーディングスタイル

## Python

Python プロジェクトのパッケージ管理には `uv` を使うこと。

```bash
uv init        # プロジェクト初期化
uv add <pkg>   # パッケージ追加
uv run <cmd>   # スクリプト実行
```

## 共通

ソースコードを作成・修正する場合は以下を守ること。

- 最小限の実装にする（不要な機能・抽象化を加えない）
- 簡潔に書く（冗長な表現を避ける）
- 読みやすさを優先する
- 意図がわかりにくい箇所には簡潔なコメントを書く（なぜそうしているかを記載する）
- 実装が大きくなる場合はファイルやモジュールへの分割を提案する
- 実装が複雑・大規模になる場合は段階的な実装を提案する
